# CLAUDE.md — ePubLangMerger

## Overview
R/Shiny application that merges two ePub files in different languages into a single bilingual ebook. Produces paragraph-level interleaved output for parallel reading.

## Tech Stack
- R (>= 3.5)
- Shiny (web UI framework)
- XML package (XHTML parsing)
- Docker (image: `geiserx/epublangmerger`)
- testthat for testing
- GitHub Actions CI

## Development
```bash
# Docker (recommended)
docker compose up -d
# Then open http://localhost:3838

# Manual R
Rscript -e "shiny::runApp('.')"

# Tests
Rscript -e "testthat::test_dir('tests')"
```

## Architecture
- `server.R` — Shiny server logic
- `ui.R` — Shiny UI definition
- `utils.R` — shared utility functions
- `script.R` — standalone CLI/batch mode for scripting
- `www/` — static web assets
- `tests/` — test suite
- `docs/` — documentation and images
- `Dockerfile` + `docker-compose.yml` — containerized deployment

## Key Rules
- Merging operates at paragraph/heading level (`<p>`, `<h1>`-`<h5>`)
- Second ePub IDs get `_2` suffix to prevent XHTML ID collisions
- Docker images use semver tags on Docker Hub (`geiserx/epublangmerger`)
- Caching: previously merged files are detected and served without reprocessing
